🔍 FAQ Informatică - Bacalaureat

1. Bazele programării

Q: Care sunt diferențele dintre compilatoare și interpretoare?
  • Compilatoarele (ex: GCC pentru C++) transformă tot codul sursă în mașinărie odată (executabil)
  • Interpretoarele (ex: Python) execută linie cu linie, fără a crea fișier executabil
  • Avantaje compilare: viteză de execuție mai mare
  • Avantaje interpretare: portabilitate, depanare mai ușoară
Q: Ce tipuri de date fundamentale cunoașteți în C++?
  • int - numere întregi (ex: -5, 0, 127)
  • float/double - numere reale (ex: 3.14)
  • char - un singur caracter (ex: 'A')
  • bool - valori logice (true/false)
  • void - tip special fără valori

2. Structuri de date

Q: Când alegem vectori și când liste înlănțuite?
Vectori când:
  • Avem nevoie de acces rapid la elemente (O(1))
  • Dimensiunea este cunoscută sau relativ constantă
Liste când:
  • Facem inserări/ștergeri frecvente (O(1))
  • Dimensiunea variază mult
Q: Ce este un arbore binar și unde se folosește?
Structură ierarhică unde fiecare nod are maximum 2 copii. Aplicații:
  • Arbori de căutare binară (BST)
  • Heap-uri (sortare și priorități)
  • Parșare expresii matematice

3. Algoritmi fundamentali

Q: Cum funcționează quicksort și care e complexitatea lui?
  • Alege un element pivot
  • Partitionează vectorul în elemente < pivot și > pivot
  • Aplică recursiv pe subvectori
Complexitate:
  • Best case: O(n log n)
  • Worst case: O(n²) (dacă pivotul e mereu minim/maxim)
  • Average case: O(n log n)
Q: Ce este programarea dinamică și când o folosim?
Tehnică care rezolvă problemele prin:
  • Descompunere în subprobleme
  • Memorarea soluțiilor subproblemelor
  • Combinarea soluțiilor
Exemple clasice:
  • Problema rucsacului
  • Șirul Fibonacci
  • Cel mai lung subșir comun

4. Operarea cu fișiere

Q: Cum citesc/scriu eficient în fișiere în C++?
#include <fstream>
using namespace std;

// Scriere
ofstream fout("fisier.out");
fout << "Text de scris" << endl;
fout.close();

// Citire
ifstream fin("fisier.in");
string linie;
while(getline(fin, linie)) {
    cout << linie << endl;
}
fin.close();
Optimizări:
  • Folosește #define cin fin pentru a redirecționa inputul
  • Pentru performanță, citește tot conținutul deodată în buffer

5. Întrebări capcana

Q: De ce 0.1 + 0.2 != 0.3 în majoritatea limbajelor?
Din cauza reprezentării în virgulă flotantă (binary floating-point), unde unele fracții zecimale nu pot fi reprezentate exact, ducând la erori de rotunjire.
Q: Ce face i+++i++ și de ce e periculos?
Comportamentul este nedefinit în C++. Depinde de compilator și poate produce rezultate diferite. Evitați expresii cu multiple modificări ale aceleiași variabile într-o singură instrucțiune.

6. Sfaturi pentru examen

Q: Cum îmi gestionez timpul în examen?
  • Alocă: 20% timp pentru citirea și înțelegerea problemelor
  • 60% pentru implementare
  • 20% pentru testare și corectare
  • Începe cu problemele pe care le consideri mai ușoare
  • Folosește comentarii pentru a marca unde ai rămas
Q: Cum verific corectitudinea algoritmilor mei?
  • Testează cu cazuri limită: vectori/siruri goale, elemente duplicate, valori minime/maxime
  • Implementează o soluție "naivă" pentru comparație
  • Verifică complexitatea timp cu un timer
